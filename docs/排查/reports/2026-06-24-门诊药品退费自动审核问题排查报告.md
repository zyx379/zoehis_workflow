## 排查方案

### 方式 A：代码逻辑分析（已产出）

**关键逻辑节点**：

1. **退药单创建**（`saveReturnDrugInfo`）：
   - 退药单初始状态固定为 `ReturnApplyStatusEnum.newApply`（"新开"）
   - 代码位置：`OutpRefundServiceImpl.java:1975`

2. **自动审核判断**（`saveNewToCheck`）：
   - 依赖系统参数 `apply_not_exec_refund_and_exec`
   - 参数值说明：
     - `"0"`：不支持（默认值）
     - `"1"`：门诊支持
     - `"2"`：住院支持
     - `"3"`：全支持
   - **如果参数为 "0"，则不会执行自动审核逻辑**

3. **原单据执行状态查询**（`getRefundApplyByExecNum`）：
   - 查询条件：`apply_sheet_status_code in ('2', '3')`
   - 状态码含义：
     - `'0'`：新开
     - `'1'`：已审核
     - `'2'`：已执行/已发送
     - `'3'`：已摆药/已发药
   - **如果原单据状态不是 '2' 或 '3'，则 `execNum = 0`，触发自动审核**

**可能断点**：

- **断点 1**：系统参数 `apply_not_exec_refund_and_exec` 未配置或为 "0"
  - 影响：不会进入自动审核分支
  - 现象：退药单保持"新开"状态

- **断点 2**：原单据状态不符合预期
  - 影响：`getRefundApplyByExecNum` 返回非 0 值
  - 现象：不触发自动审核

```sql
-- ============================================================
-- 现场排查脚本 | 门诊药品退费自动审核问题
-- 项目：【待确认】| 生成日期：2026-06-24
-- 使用前请替换：:EVENT_NO :PATIENT_ID :APPLY_NO
-- 约束：仅 SELECT；禁止在生产库直接 UPDATE/DELETE
-- ============================================================

-- ---------- 0. 系统参数核查 ----------
-- 验证：关键系统参数是否配置正确
-- 预期：apply_not_exec_refund_and_exec 应为 '1' 或 '3'（门诊支持）
SELECT param_english_name, param_value, org_id, param_desc
FROM com_sys_param
WHERE param_english_name IN (
    'apply_not_exec_refund_and_exec',      -- 单据未执行退费即审核
    'dept_return_apply_check_limit',       -- 退费审核模式
    'refund_apply_equal_exec_auto_check'   -- 开单科室等于执行科室自动审核
)
ORDER BY param_english_name;

-- ---------- 1. 假设 H1：系统参数配置 ----------
-- 验证：apply_not_exec_refund_and_exec 参数值
-- 预期：值应为 '1'（门诊支持）或 '3'（全支持）
-- 异常含义：如果为 '0' 或空，则不会触发自动审核
SELECT param_value, 
       CASE 
           WHEN param_value = '0' THEN '不支持自动审核'
           WHEN param_value = '1' THEN '门诊支持'
           WHEN param_value = '2' THEN '住院支持'
           WHEN param_value = '3' THEN '全支持'
           ELSE '未配置或值异常'
       END AS param_desc
FROM com_sys_param
WHERE param_english_name = 'apply_not_exec_refund_and_exec';

-- ---------- 2. 假设 H2：原单据状态 ----------
-- 验证：问题退费单对应的原单据状态
-- 预期：未摆药的原单据状态应为 '0'（新开）或 '1'（已审核），不应是 '2' 或 '3'
-- 异常含义：如果原单据状态是 '2' 或 '3'，则 getRefundApplyByExecNum 返回非 0，不触发自动审核
SELECT 
    ocd.apply_no AS 原单据号,
    ocd.charge_detail_no AS 费用明细号,
    ocd.event_no AS 挂号号,
    ocd.patient_id AS 患者ID,
    ocd.clinic_item_code AS 项目代码,
    ocd.clinic_item_name AS 项目名称,
    ocd.drug_flag AS 药品标识,
    ocd.apply_sheet_status_code AS 单据状态,
    ocd.report_status_code AS 报告状态,
    ocd.exec_dept_code AS 执行科室,
    ocd.charge_time AS 扣费时间,
    CASE 
        WHEN ocd.apply_sheet_status_code IN ('2', '3') THEN '已执行/已摆药'
        WHEN ocd.apply_sheet_status_code = '1' THEN '已审核'
        WHEN ocd.apply_sheet_status_code = '0' THEN '新开'
        ELSE '状态异常'
    END AS 状态说明
FROM zoeapply.v_app_outp_apply ocd
WHERE ocd.event_no = :EVENT_NO
  AND ocd.patient_id = :PATIENT_ID
  AND ocd.drug_flag = '1'  -- 药品
ORDER BY ocd.charge_time DESC;

-- ---------- 3. 假设 H3：视图数据一致性 ----------
-- 验证：视图 v_app_outp_apply 与源表数据是否一致
-- 预期：视图数据应与源表一致
-- 异常含义：如果不一致，可能是视图定义问题或数据同步问题
SELECT '视图' AS 数据来源, apply_no, apply_sheet_status_code, report_status_code
FROM zoeapply.v_app_outp_apply
WHERE apply_no = :APPLY_NO
UNION ALL
SELECT '源表' AS 数据来源, apply_no, apply_sheet_status_code, report_status_code
FROM zoeapply.app_outp_apply_sheet
WHERE apply_no = :APPLY_NO;

-- ---------- 4. 退费单状态查询 ----------
-- 验证：问题退费单的当前状态
-- 预期：未摆药的退费单应为"审核通过"状态
-- 异常含义：如果状态为"新开"，说明自动审核未触发
SELECT 
    rm.apply_no AS 退费单号,
    rm.event_no AS 挂号号,
    rm.patient_id AS 患者ID,
    rm.refund_apply_status_code AS 退费状态,
    rm.apply_time AS 开单时间,
    rm.check_operated_time AS 审核时间,
    rm.apply_dept_code AS 开单科室,
    rm.check_dept_code AS 审核科室,
    CASE 
        WHEN rm.refund_apply_status_code = '0' THEN '新开'
        WHEN rm.refund_apply_status_code = '1' THEN '已审核'
        WHEN rm.refund_apply_status_code = '2' THEN '已执行'
        ELSE '状态异常'
    END AS 状态说明
FROM zoeapply.app_refund_master rm
WHERE rm.apply_no = :APPLY_NO;

-- ---------- 5. 退费细表与原单据关联 ----------
-- 验证：退费单关联的原单据执行状态
-- 预期：未摆药的原单据不应有 '2' 或 '3' 状态
SELECT 
    rd.apply_no AS 退费单号,
    rd.apply_sub_no AS 退费细表序号,
    rd.orig_apply_no AS 原单据号,
    rd.item_code AS 项目代码,
    rd.item_name AS 项目名称,
    v.apply_sheet_status_code AS 原单据状态,
    v.report_status_code AS 原报告状态,
    CASE 
        WHEN v.apply_sheet_status_code IN ('2', '3') THEN '已执行/已摆药'
        WHEN v.apply_sheet_status_code = '1' THEN '已审核'
        WHEN v.apply_sheet_status_code = '0' THEN '新开'
        ELSE '状态异常'
    END AS 原单据状态说明
FROM zoeapply.app_refund_detail rd
LEFT JOIN zoeapply.v_app_outp_apply v ON rd.orig_apply_no = v.apply_no
WHERE rd.apply_no = :APPLY_NO;

-- ---------- 6. 字典项核查 ----------
-- 验证：单据状态字典值是否正确
-- 预期：'2' 和 '3' 应对应已执行/已摆药状态
SELECT dict_name, value_code, value_name, data
FROM zoedict.dic_basic_dict
WHERE dict_name = 'REPORT_STATUS_DICT'
ORDER BY value_code;

-- ---------- 7. 综合排查（推荐优先执行） ----------
-- 验证：一次性核查关键信息
SELECT 
    '系统参数' AS 检查项,
    param_english_name AS 参数名,
    param_value AS 参数值,
    CASE 
        WHEN param_english_name = 'apply_not_exec_refund_and_exec' AND param_value IN ('1', '3') THEN '配置正确'
        WHEN param_english_name = 'apply_not_exec_refund_and_exec' THEN '配置异常，应为1或3'
        ELSE '待确认'
    END AS 检查结果
FROM com_sys_param
WHERE param_english_name IN ('apply_not_exec_refund_and_exec', 'dept_return_apply_check_limit')
UNION ALL
SELECT 
    '退费单状态' AS 检查项,
    TO_CHAR(apply_no) AS 参数名,
    refund_apply_status_code AS 参数值,
    CASE 
        WHEN refund_apply_status_code = '1' THEN '已审核（正常）'
        WHEN refund_apply_status_code = '0' THEN '新开（异常）'
        ELSE '状态异常'
    END AS 检查结果
FROM zoeapply.app_refund_master
WHERE apply_no = :APPLY_NO;
```
