# 返工需求流程（Trae 分析 → Cursor 实现）

> 适用：已交付需求测试不通过、小范围修补。省 token：Trae 只分析，Cursor 只实现。

## 流程图

```
测试反馈
  ↓
Trae：复制 dev/返工需求prompt.template → dev/返工需求prompt
  ↓ Step 0–4（见 docs/prompt_external.md）
  ↓ 搜 docs/memory + 写【分析结果】，状态→已分析
  ↓ 可选：docs/memory/short-term/{禅道号}-rework.md
  ↓
Cursor：@dev/返工需求prompt
  ↓ 已分析 → 最小改码 → Step 8 审查
  ↓ 人审「审查通过，提交并发布」
  ↓ push master → merge 项目分支 → tag+1
  ↓ Step 12 更新 case / 删 short-term
```

## 分工

| 阶段 | 工具 | 做 | 不做 |
|------|------|-----|------|
| 分析 | Trae | 记忆库检索、根因、文件定位、修复方案 | 改代码、git |
| 实现 | Cursor | 按分析改码、审查、交付 | 重复全量需求分析 |

## Trae 开场（复制）

```text
请按 docs/prompt_external.md Step 0–4；禁止改代码/git。
任务：返工分析，产出写入 dev/返工需求prompt【分析结果】，状态改已分析。
先读 docs/memory/index.md 找关联 case。

（粘贴下方【禅道&测试反馈】块）
```

## Trae 输入块（最小）

```text
【禅道号】202858
【项目】【漳州二院】
【测试反馈】只保存了部位，方位没有保存
【关联 case】docs/memory/cases/2026-06-operationApply-preopDiag-sync.md
```

## Cursor 开场（复制）

```text
@dev/返工需求prompt
按 workflow Trivial 路径：已分析→实现→审查→等人审
```

## 交付注意

- commit 标题与**首次交付同项目关键词**（【漳州二院】→ release-1.166；【漳州市医院】→ release-1.168）
- 返工可在原 case 追加「返工记录」，不必新建 case（除非域完全不同）
- 参数 jsonl 仍单独 commit

## 与全新需求对比

| | 全新需求 Trae | 返工 Trae |
|--|--------------|-----------|
| 记忆 | index + short-term 新建 | **优先已有 case** |
| 产出 | short-term + 交接块 | **dev/返工需求prompt** |
| Cursor 起点 | Step 5 spec | **直接 Step 6**（Trivial） |
